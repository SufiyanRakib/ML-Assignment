🩺 Heart Disease Prediction

Benchmarking ML Classifiers with Real Healthcare Data

Python 3.8+ Scikit-Learn Jupyter Notebook CDC Dataset

📌 Project Overview

Predicting heart disease is one of the most critical applications of machine learning in healthcare. This project benchmarks three distinct classification approaches on the CDC BRFSS dataset containing 319,795 medical records.

Classification Paradigms Compared:

  • Distance-based: K-Nearest Neighbors (KNN)
  • Boundary-optimizing: Support Vector Machines (SVM) with Linear & RBF Kernels
  • Probabilistic: Gaussian Naive Bayes (GNB)
⚠️ Key Challenge: The dataset has severe class imbalance (only 9.2% heart disease cases), requiring careful metric selection and evaluation strategies.

🩺 Clinical Recommendations

Recommended Two-Stage Approach:

Stage 1: Triage Screening

Use Naive Bayes

Recall: 73.24% - Catches 3 out of 4 disease cases

Stage 2: Confirmation

Use SVM (Linear)

Precision: 70.65% - High accuracy confirmation

💡 Why Recall Over Precision in Medical Screening?

False Negatives (missing disease) are 10x more dangerous than False Positives (unnecessary follow-up tests). In healthcare, better to over-screen than under-screen.

⚙️ Data Preprocessing Pipeline

  • Target Encoding: HeartDisease: Yes → 1, No → 0
  • One-Hot Encoding: Applied to 9 categorical features
  • Train/Test Split: 80%/20% with stratification
  • Feature Scaling: StandardScaler (except for Naive Bayes)
